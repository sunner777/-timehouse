## 2. 数据库设计

### 2.1 MySQL 数据库（用户、家庭组、权限）

#### 用户表 (users)
```sql
CREATE TABLE users (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
  phone VARCHAR(20) NOT NULL UNIQUE COMMENT '手机号',
  password VARCHAR(255) NOT NULL COMMENT '密码（bcrypt加密）',
  nickname VARCHAR(50) DEFAULT '' COMMENT '昵称',
  avatar VARCHAR(500) DEFAULT '' COMMENT '头像URL',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_phone (phone),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

#### 家庭组表 (families)
```sql
CREATE TABLE families (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '家庭组ID',
  name VARCHAR(100) NOT NULL COMMENT '组名',
  owner_id BIGINT UNSIGNED NOT NULL COMMENT '群主ID',
  invite_code CHAR(6) NOT NULL UNIQUE COMMENT '邀请码',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_owner_id (owner_id),
  INDEX idx_invite_code (invite_code),
  FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='家庭组表';
```

#### 家庭组成员表 (family_members)
```sql
CREATE TABLE family_members (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '成员ID',
  family_id BIGINT UNSIGNED NOT NULL COMMENT '家庭组ID',
  user_id BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
  role ENUM('owner', 'admin', 'member', 'guest') DEFAULT 'member' COMMENT '角色',
  permissions JSON DEFAULT NULL COMMENT '权限列表["view","edit","delete","manage"]',
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '加入时间',
  UNIQUE KEY uk_family_user (family_id, user_id),
  INDEX idx_user_id (user_id),
  FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='家庭组成员表';
```

#### 权限说明
| 角色 | 默认权限 | 说明 |
|------|----------|------|
| owner | ["view","edit","delete","manage"] | 群主，拥有所有权限 |
| admin | ["view","edit","delete"] | 管理员，可编辑和删除 |
| member | ["view","edit"] | 成员，可查看和编辑 |
| guest | ["view"] | 访客，仅可查看 |

### 2.2 MySQL 照片表（实际实现）

> 注：原方案计划用 MongoDB 存照片元数据，实际 V1.0/V2.0 均使用 MySQL 统一存储。MongoDB 和 Redis 已连接但不承载业务数据，预留后续使用。

#### 照片表 (photos)
```sql
CREATE TABLE photos (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '照片ID',
  user_id BIGINT UNSIGNED NOT NULL COMMENT '上传者ID',
  family_id BIGINT UNSIGNED DEFAULT NULL COMMENT '家庭组ID（可选）',
  url VARCHAR(2000) NOT NULL COMMENT '原图URL（TOS预签名）',
  thumbnail_url VARCHAR(2000) DEFAULT NULL COMMENT '缩略图URL',
  file_name VARCHAR(255) NOT NULL COMMENT '文件名',
  size BIGINT UNSIGNED NOT NULL COMMENT '文件大小（字节）',
  content_type VARCHAR(50) NOT NULL COMMENT 'MIME类型',
  taken_at TIMESTAMP NULL COMMENT '拍摄时间',
  location VARCHAR(255) DEFAULT '' COMMENT '拍摄位置',
  tags JSON DEFAULT NULL COMMENT '标签数组',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '上传时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_user_id (user_id),
  INDEX idx_family_id (family_id),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='照片表';
```

## 4. 错误码规范

### 4.1 错误码区间
| 区间 | 模块 | 说明 |
|------|------|------|
| 1000-1999 | 认证模块 | 用户注册、登录、密码等 |
| 2000-2999 | 照片模块 | 照片上传、删除、查询等 |
| 3000-3999 | 家庭组模块 | 家庭组创建、管理等 |
| 4000-4999 | 人脸识别模块 | 人脸识别、人物管理等 |
| 9000-9999 | 系统级错误 | 服务器错误、参数错误等 |

### 4.2 通用错误码
| code | message | 说明 |
|------|---------|------|
| 9001 | 参数缺失 | 必填参数未提供 |
| 9002 | 参数格式错误 | 参数格式不符合要求 |
| 9003 | 未授权 | Token无效或已过期 |
| 9004 | 无权限 | 当前用户无此操作权限 |
| 9005 | 资源不存在 | 请求的资源不存在 |
| 9006 | 资源已存在 | 资源冲突 |
| 9007 | 操作失败 | 业务操作失败 |
| 9008 | 服务器错误 | 内部服务器错误 |

### 5.1 Key命名规范
```
{prefix}:{module}:{identifier}:{attribute}

示例:
- auth:token:{userId}           # 用户Token
- user:profile:{userId}         # 用户信息缓存
- family:members:{familyId}     # 家庭组成员缓存
- family:permissions:{familyId}:{userId}  # 用户在家庭组中的权限
- photo:cache:{photoId}         # 照片信息缓存
- person:faces:{personId}       # 人物人脸数据缓存
```

### 5.2 缓存策略
| Key类型 | TTL | 说明 |
|---------|-----|------|
| Token | 7天 | 与JWT过期时间一致 |
| 用户信息 | 1小时 | 频繁访问的用户数据 |
| 家庭组成员 | 30分钟 | 组成员列表 |
| 权限缓存 | 15分钟 | 用户权限数据 |
| 照片信息 | 2小时 | 照片元数据 |
| 识别结果 | 永不过期 | 人脸识别结果，重要数据 |

### 5.3 缓存更新策略
- **主动更新**：数据变更时立即更新缓存
- **被动失效**：使用Cache-Aside模式，读时填充，写时失效
- **定时刷新**：热点数据定时更新，保证数据新鲜度

## 6. 核心业务流程

### 6.1 照片上传流程（实际实现）

```
1. 前端请求后端获取TOS上传签名
   POST /api/v1/photos/tos-upload-signature
   请求参数: { fileName, contentType }
   返回: { uploadUrl, objectKey }

2. 前端直接上传文件到TOS
   http.put(uploadUrl, body: fileBytes, headers: { 'Content-Type': contentType })

3. 前端通知后端保存照片元数据
   POST /api/v1/photos/upload
   请求参数: { url, thumbnailUrl, fileName, size, contentType, takenAt, location, tags, familyId }

4. 后端处理
   - 清理URL（去反引号和空格）
   - 创建照片记录（MySQL photos 表）
   - 返回 photoId

5. 前端选择可见范围（UploadFAB 组件）
   - 弹出共享组选择器
   - 对每个已上传照片调用 PUT /api/v1/photos/:id/family
   - 更新照片归属到选中的家庭组（或仅自己可见）
```

### 6.2 家庭组权限验证流程
```
1. 用户发起操作请求
   DELETE /api/v1/families/{familyId}/photos/{photoId}

2. 后端权限验证
   - 从Token获取userId
   - 检查Redis缓存: family:permissions:{familyId}:{userId}
   - 缓存未命中，从数据库查询
   - 验证权限是否包含"delete"
   - 更新Redis缓存

3. 执行操作或返回无权限错误
```

### 6.3 人脸识别流程
```
1. 照片上传完成，触发识别任务

2. 识别任务入队
   - 将任务加入消息队列
   - 记录任务状态为pending

3. Worker进程处理
   - 取出任务
   - 调用火山引擎人脸识别API
   - 获取人脸检测结果

4. 结果处理
   - 提取人脸特征
   - 与现有人物库比对
   - 创建新人物或关联现有人物
   - 更新照片元数据（faces, faceRecognized: true）
   - 更新人物集合（faceIds, faceCount）

5. 识别失败处理
   - 重试最多3次（指数退避）
   - 最终失败记录错误日志
   - 状态标记为recognize_failed
```

## 7. 安全措施

### 7.1 认证安全
- **密码加密**：使用bcrypt，cost factor 12
- **Token生成**：HS256算法，7天有效期
- **Token刷新**：使用refreshToken，30天有效期

### 7.2 接口安全
- **签名验证**：关键接口使用API签名
- **限流**：每IP每分钟100次请求
- **参数校验**：使用joi进行严格校验

### 7.3 数据安全
- **TOS访问**：使用预签名URL，限制上传时间
- **敏感数据**：手机号脱敏显示
- **日志脱敏**：密码、Token不记录日志
