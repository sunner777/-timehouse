# 「时光家」App 项目规划

## 1. 项目概述

「时光家」是一款家庭相册App，专注于照片管理、共享和智能识别，帮助用户更好地记录和分享家庭时光。

### 核心价值
- **安全存储**：使用火山引擎TOS存储照片，安全可靠
- **智能识别**：本地人脸识别，自动按人物分类
- **家庭共享**：创建家庭组，邀请成员共同管理照片
- **跨设备同步**：多设备数据一致，随时随地查看

## 2. 核心记忆

### 2.1 技术方案要求
- **中国大陆用户访问最佳性能**：所有技术方案均考虑中国大陆用户访问最佳性能
- **沙箱操作**：判断如果在沙箱外操作更合适的行动点，除非要求在沙箱内完成操作，否则不要自己在沙箱内操作，要告知用户协助在沙箱外操作，并告知操作结果
- **错误处理**：如果遇到相同的错误信息，修改多次均没有效果，则需要思考是不是修改思路有问题，也许需要用户在沙箱外帮助才能修复问题，如果需要，则告知用户

## 3. 核心功能 Summary

### 3.1 账号系统
- **注册/登录**：手机号注册、密码登录、验证码登录
- **设备同步**：同一账号在多设备上的数据同步
- **密码管理**：支持密码重置、修改

### 3.2 共享功能
- **家庭组**：创建家庭组，邀请成员加入
- **共享相册**：组内成员可共同查看、上传、删除照片
- **权限控制**：可设置成员权限（查看/编辑/管理）
- **邀请机制**：通过手机号或邀请码邀请

### 3.3 人脸识别
- **本地识别**：在设备本地进行人脸识别，保护隐私
- **人物分类**：自动按人物分类照片，类似iPhone Photos
- **智能精选**：基于人脸识别生成人物相册
- **跨设备同步**：
  - 识别结果以用户/家庭组为维度同步
  - 不重复识别，节省性能
  - 多设备之间保持一致
  - 识别结果与照片一起存储，所有成员看到相同的分类

### 3.4 照片管理
- **上传**：支持批量上传，每张≤5M，最多10张/次
- **存储**：原图存储在火山引擎TOS，本地缓存缩略图
- **浏览**：按日期、人物分类浏览
- **删除**：支持批量删除
- **懒加载**：优化性能，避免卡顿

## 4. 技术架构

### 4.1 前端
- **框架**：Flutter (Dart)
- **平台**：Android + iOS（后续可扩展）
- **核心库**：
  - `tflite_flutter`（本地人脸识别）
  - `dio`（网络请求）
  - `sqflite`（本地数据库）

### 4.2 后端（火山引擎）
- **认证服务**：用户注册/登录/管理
- **云数据库**：
  - **关系型数据库**（火山引擎 RDS MySQL）：存储用户、家庭组、权限等结构化数据
  - **文档型数据库**：存储照片元数据等大量小记录
- **云函数**：处理业务逻辑
- **API 网关**：统一接口管理

### 4.3 存储（火山引擎 TOS）
- **photos/**：存储所有照片文件
- **recognition/**：存储识别结果 JSON

### 4.4 安全措施
- **数据传输**：HTTPS加密、API签名
- **存储安全**：TOS访问控制、服务端加密
- **认证安全**：密码策略、JWT Token
- **权限控制**：RBAC权限模型、操作审计
- **隐私保护**：本地识别、数据最小化
