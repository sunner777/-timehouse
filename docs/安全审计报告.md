# 时光家 V1.0 安全审计报告

审计日期：2026-06-13
审计范围：全项目（后端、前端、服务器配置）

---

## 🔴 Critical — 必须立即修复

### 1. 添加成员 API 泄露目标用户手机号

**文件**：`backend/src/services/familyService.js:243`

```js
// ❌ 当前代码
return {
  userId: user.id,
  nickname: user.nickname,
  phone: user.phone,   // <-- 泄露！调用者能看到被添加人的完整手机号
  role
};
```

**风险**：通过「添加成员」功能，群主/管理员可以批量查询任意手机号是否注册，且拿到完整手机号。

**修复**：去掉 `phone` 字段，或只返回脱敏后的手机号（如 `153****4275`）。

---

## 🟠 High — 应尽快修复

### 2. JWT Token 无吊销机制

**文件**：`backend/src/services/authService.js`

- JWT 有效期 7 天，签发后无法主动吊销
- Token 一旦泄露（如手机丢失、XSS、日志泄露），攻击者拥有完整 7 天访问权限
- 没有 Token 黑名单或刷新机制

**修复方案**：
- a) 缩短 JWT 有效期到 24 小时 + 加 Refresh Token
- b) 在 Redis 中维护一个 Token 黑名单（登出/密码变更时加入），auth 中间件检查

### 3. 错误信息泄露实体存在性（用户枚举）

**文件**：`backend/src/services/photoService.js:96-112`

```js
// 两个不同的错误返回同一个 "照片不存在"
// 但时间延迟不同（DB 查询 vs DB 查询 + 成员检查）
```

**风险**：攻击者可以通过响应时间差异推断照片是否存在、用户是否在某个家庭组中。

**修复**：统一所有未授权访问返回相同错误，确保响应时间一致。

### 4. 照片预签名 URL 有效期过长

**文件**：`backend/src/services/tosService.js:42`

- 当前 3600 秒（1 小时）
- API 响应被截获后，攻击者可在 1 小时内任意下载照片

**修复**：缩短到 300 秒（5 分钟），前端刷新时重新生成即可。

---

## 🟡 Medium — 中期改进

### 5. 无操作审计日志

**当前状态**：没有记录任何人查看了哪张照片、修改了哪些权限。

**风险**：
- 隐私泄露后无法溯源
- 无法回答"谁看过我的照片"这类基本的隐私问题
- 违反数据隐私最小化原则

**修复**：至少记录以下操作到数据库或日志文件：
- 谁 + 什么时间 + 查看了哪张照片
- 谁 + 什么时间 + 修改了谁的权限
- 谁 + 什么时间 + 删除/上传了照片

### 6. 全局速率限制是 IP 级别的

**文件**：`backend/src/middleware/rateLimiter.js`

- 当前：100 次/15 分钟/IP
- SMS 接口有独立的 Redis 限制（✅ 好）
- 但其他接口的 IP 级限制容易被代理绕过

**修复**：对认证接口额外加入按 userId 的速率限制。

### 7. FamilyMember 查询不必要地选择了手机号

**文件**：`backend/src/models/mysql/FamilyMember.js:50-58`

```sql
SELECT fm.*, u.nickname, u.avatar, u.phone  -- phone 不需要！
FROM family_members fm
INNER JOIN users u ON fm.user_id = u.id
```

**修复**：去掉 `u.phone` 从 SELECT 中，虽然目前 API 层没暴露但万一后续有人改了映射就直接泄露。

### 8. Node.js 请求体大小无限制

**当前**：只有 Nginx 的全局 `client_max_body_size 10m`

**风险**：绕过 Nginx 直连 Node 时（如本地攻击），可发送超大请求体导致 OOM。

**修复**：在 Express 中添加 `express.json({ limit: '10mb' })`

### 9. 云服务凭证明文存储

**文件**：`backend/.env.production`

- TOS AK/SK、SMS AK/SK、JWT Secret 都是明文
- 拿到服务器 SSH 即拿到所有凭据

**风险**：服务器被入侵后，可消费 SMS 额度、访问 TOS 存储桶全部文件。

**修复**：
- SSH 禁用密码登录，仅密钥认证
- 定期轮换 AK/SK
- 考虑使用环境变量注入替代 `.env` 文件

## 🟢 Low — 优化建议

### 10. 手机号正则未覆盖所有中国号段

**文件**：`backend/src/services/smsService.js:19`

```js
/^1[3-9]\d{9}$/
```

**缺失号段**：162、165、167、170、171 等虚拟运营商号段。

**修复**：改为 `/^1[3-9]\d{9}$/` → `/^1\d{10}$/` 或维护完整的号段白名单。

### 11. JWT Secret 无轮换机制

- 当前 Secret 固定不变
- 一旦泄露，所有已签发的 Token 立即失效需全部重新登录

**修复**：定期轮换 JWT Secret，或使用密钥版本管理。

### 12. 僵尸依赖

**`package.json`**：
- `mongoose` — MongoDB 已不再使用
- `winston` — 日志用的是 `console.log` 而非 winston

**修复**：删除无用的依赖，减少攻击面。

## 数据流隐私地图

```
用户手机号：
  存储：users.phone (MySQL)
  传输：登录请求 → API → SMS 平台
  暴露点：❌ addMemberByPhone 返回完整手机号

用户照片：
  存储：火山引擎 TOS（私有 Bucket）
  传输：预签名 URL（HTTPS）
  访问控制：✅ 家庭组成员校验
  泄露风险：⚠️ 预签名 URL 有效期 1 小时

JWT Token：
  存储：客户端 SharedPreferences（明文）
  传输：HTTP Header Bearer
  有效期：7 天 ⚠️ 偏长，无法吊销
```
